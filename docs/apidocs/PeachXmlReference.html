Documentation Notes on Peach XML

Peach XML documents are parsed top down.  This means you cannot reference items that have not appeared yet in the XML.  Includes and Imports should appear first, then Templates and finally Data elements.

Importing Defaults

All Peach XML files should include defaults.xml.  Defaults.xml configures the required paths and performs the basic default imports of modules.  This is typically site specific information.  Defaults.xml should never be bundled with your template, always the use sites version.

PythonPath

The PythonPath element allows adding a path to sys.path.  This is the path used by Python to locate modules.

Attribute Description
path Path to append to search path
   

 

Include

The Include element allows for including other Peach XML files into the current context or namespace.

Attribute Description
ns Namespace prefix.  A prefix applied to any element being referenced in this XML file.  The format is "ns:element".  If the ns attribute is set to "foo" then one might say "foo:Template1" to reference Template1 element.
src The URI of the Peach XML file to include.  By default we support file and http.

 

Import

The Import element allows for importing python modules into the current namespace.

Attribute Description
from Just like the from N import X syntax.
import Just like the from N import X syntax.

 

Template

The Template element describes the format of data and the relationship of it's fields.  Templates can build off other Templates.

Attribute Description
name Name of template
ref [OPTIONAL] Causes current element to "extend" referenced element.  Elements must be of same type with the exception of Template and Blocks which may be mixed.

 

Common

These are common properties of all Template child elements.

Attribute Description
   
name [OPTIONAL] Unique (to template) name that is used to reference the element.
minOccurs [OPTIONAL] Indicated the minimum number of times this block must occur in a valid data generation.  When minOccurs and maxOccurs are not indicated, minOccurs and maxOccurs default to 1.  If maxOccurs is indicated then minOccurs defaults to 0.
maxOccurs [OPTIONAL] Indicates the maximum number of times this block can occur in a valid data generation.  When minOccurs is specified, but maxOccurs is omitted it defaults to unlimited.
value [OPTIONAL] Default value.  Default values can be later overridden using a Data element.
valueType [OPTIONAL] How is default value represented.  Valid values are "literal" or "hex".  Defaults to "literal".

 

Block

Attribute Description
alignment [OPTIONAL] Ala structure alignment.  Value raised to the power of 2.  In theory 1, 2, 4, 8, or 16 are the allowed values.
ref [OPTIONAL] Causes current element to "extend" referenced element.  Elements must be of same type with the exception of Template and Blocks which may be mixed.

String

Element representing a character string.  Currently strings are single or double bytes (char or wchar) with a possible trailing null terminator.

Attribute Description
type [OPTIONAL] Type of string to be generated.  Valid values are: char, wchar, or utf8
length [OPTIONAL] Length for fixed length strings.  If the string generated is shorter then this it is padded with the padCharacter.
nullTerminated [OPTIONAL] Is the generated string null terminated.  Valid values are true or false.
value [?][OPTIONAL] Default value for string
padCharacter [OPTIONAL] Character to pad fixed length strings with.  Default is NULL.

 

Number

This element represents a binary number that is in little or big endian byte order.  If a numerical string is wanted, use the String element instead.

Attribute Description
signed [OPTIONAL] Indicates if number is signed.  Defaults to true.
size Size in bits of number.  Valid values are 8, 16, 32, 64.
endian [OPTIONAL] Indicates byte order of number.  Defaults to little.

 

Blob

A static binary value.

Attribute Description
length [OPTIONAL] Maybe this required?  length in bytes
   
value Default value for Blob
padValue [OPTIONAL] Padding value for short data.  Defaults to NULL (0x00).

 

Flags

The flags element defines a set of bit flags.  The Flags element must have one or more Flag elements and children.

Attribute Description
length Length in bits.  Valid values are 8, 16, 32, and 64
endian [OPTIONAL] Little or big.  Defaults to little

 

Flag

A bit flag.  The common attributes of minOccurs and maxOccurs are not relevant in this context.

Attribute Description
position Position (zero based) this flag starts at in bits
length Length of flag in bits.  position+length must not be larger then length specified in Flags parent element.

 

Relation

The Relation element must be a sub child of Block, String, Number, Flag, or Blob.  This element indicates a relation-ship between two elements.  Examples of relation ships would be one element indicating the size of another element.  Or a flag that indicates an optional block is there or not.

Attribute Description
type Type of relation. Valid values: sizeof
of Name of another element

Value

Specifies a default value for an element of type String, Number, Flag or Blob.

Attribute Description
   
value Default value

Field

Attribute Description
name Field in template this data is a default for.  The name takes the form "ns:template.element.subelement"  ns (namespace) may not be required.
value Value of field.  This is required.
valueType [OPTIONAL] Can be "string" or "hex".  Defaults to "string".  When hex, we expect value to be a hex string.

 

 

Generator

The Generator element is a valid child of Number, String, Flags, and Blob.  Generator elements can have zero or more Param elements as children.  The Param elements provide constructor arguments.

Note: If you require a complex Generator setup, create a Generator that contains the complex setup and import the module/class using the Import element then reference it here.

Attribute Description
class Generator class
   

Param

Param elements provide constructor arguments for the class specified by the Generator parent element.  It is possible to pass python types like arrays by specifying valueType as literal and providing python literal as value.  e.g. "[ 1, 2, 3, 4 ]" would be an array.

Attribute Description
name Name of argument
value Argument value
valueType string, literal, hex.  Defaults to string.
